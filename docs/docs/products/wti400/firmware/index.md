# Firmware Overview

This section describes WTI400 firmware design intent, configuration approach, and change tracking.

Firmware scope includes acquisition and processing of legacy wind transducer signals, protocol output on marine networks, and operational status behavior through local indicators and app-assisted workflows.

## Contents

* *[Architecture](architecture.md)* - runtime structure, module boundaries, and processing responsibilities;
* *[Configuration](config.md)* - build-time and runtime parameters used to adapt the firmware to hardware revisions and transducer variants;
* *[Change Log](changelog.md)* - document-level change history for firmware documentation.
