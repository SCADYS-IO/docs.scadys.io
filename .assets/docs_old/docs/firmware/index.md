# Firmware

This section documents the firmware architecture and software configuration of the MDD400. It includes an overview of the runtime system, key modules, and configuration structures that support device operation, diagnostics, communication, and user interaction.

* *[Architecture](architecture.md)*: Describes the structure of the firmware, including tasks, interrupt handling, state management, and system services. Covers startup sequence, peripheral drivers, and task scheduling.

* *[Configuration](config.md)*: Details the storage and processing of system configuration data, including persistent settings, runtime parameters, and default profiles. Explains how configuration is loaded, saved, and validated during operation.

* *[Appendices](appendices.md)* and *[Change Log](changelog.md)*: Supplementary notes and version history for firmware development and field deployment.

This section provides a reference for developers working on firmware updates, debugging, or feature extensions, and outlines the operational flow of the embedded software stack.
